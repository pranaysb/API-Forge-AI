# Architecture

API Forge AI is built using a directed acyclic graph (DAG) via [LangGraph](https://python.langchain.com/v0.1/docs/langgraph/). It orchestrates multiple LLMs to generate, test, and repair Python SDKs generated from OpenAPI schemas.

## High-Level Flow

1. **Upload & Parse**: The user uploads an OpenAPI schema. The backend extracts the endpoints, paths, methods, and the `base_url`.
2. **LangGraph State Machine**: A new `IntegrationJob` is dispatched to the background. The job initializes an `AgentState` containing the schema and endpoints.

### The Nodes

- **Planner (`planner_node`)**: Initializes the execution trace and determines the topological order of endpoints to test.
- **SDK Validator (`sdk_validator_node`)**: Calls out to `sdk_builder.py` to statically generate the initial `client.py` and `models.py` files using LLMs. It then checks these files for basic AST and structural consistency.
- **Schema Validator (`schema_validator_node`)**: Writes a raw `httpx` script to fetch a live/mocked payload from the API and asserts that the data conforms to the Pydantic models generated by the SDK Validator. If the model is wrong, it marks `SCHEMA_FAILED` and routes to the Diagnoser.
- **Coder (`coder_node`)**: Writes a comprehensive, self-contained `pytest` script (typically using `httpx.MockTransport`) that instantiates the generated SDK client and tests a specific endpoint. 
- **Executor (`executor_node`)**: Spawns a secure subprocess to run the test script outputted by the Coder. It captures STDOUT, STDERR, and the exit code. If the test fails, it routes to the Diagnoser.
- **Diagnoser (`diagnoser_node`)**: Acts as the "Self-Healing" mechanism. It reviews the Execution logs, the generated SDK source code, and the test script. Depending on the error, it patches the SDK files in-memory or feeds mutation instructions back to the Coder. It guarantees safety against infinite loops via a hard attempt limit (5 retries) and strict schema validation.

## Deployment

The system is split into two deployable artifacts:
- **Backend (FastAPI)**: Serves the REST API, runs the LangGraph workflows, and handles SQLite/PostgreSQL connections using SQLAlchemy.
- **Frontend (Next.js)**: Provides the dashboard to view job execution timelines, debug logs, and download the final built ZIP artifacts.
